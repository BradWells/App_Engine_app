Experience

Digi International Internship

In the summer of 2013, I worked in the Systems Assurance section of Digi International. I primarily worked with their Git repository of tests, which had become too big to handle, and they wanted to make it more modular for each of their teams. I worked to split the Git repository into two modular ones, organized in such a way that more modular git repostitories, one for shared test environment and one repository containing all of the individual projects' tests. This change was made so that the latter repository could be broken down further into repositories for the tests of each individual project, to keep them seperate. Additionally, the test environment repository could be installed as a python egg package in the user's python enviornment, making the process even cleaner.

One problem with this new scheme is that the package name in all of the test code changed. We looked for fixes or work-arounds so we would not have to refactor the old test codes, but after some analysis we decided that the best course of action would be to change the package names in all of the test code. Because of this, one sub-project was to write a sed script in Bash that would replace the old name with the new, but change nothing else. I set up my own repo for my changes, because I didn't want to risk polluting the main repository with name changes, and so I could also triple check that the sed script was working correctly and that all of the tests still passed.

In addition to the main project, I also took on a side-project of updating our python Django web server from using an old API that used XML-RPC to send commands to the server in XML into one that used RESTful JSON requests to url's that ran code stored on the server itself. It returned JSON that was much easier to manage and gave much more relevant, clean data to be used by our other systems.
